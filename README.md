# 법률-시행령 형식 기준 F분류 탐색기

804 전체 기준과 557 의견조회 기준을 분리해 같은 UI에서 전환할 수 있는 정적 탐색기입니다.

**GitHub Pages:** <https://hosungseo.github.io/local-delegation-field-admin/>

사무 성격 판단은 제외하고 **법률-시행령 형식**만 기준으로 보는 정적 탐색기입니다. 화면 상단에서 `557 의견조회 기준`과 `804 전체 기준`을 전환합니다.

## 기준 데이터

### 557 의견조회 기준

- 파일: `app/data/delegation-557.json`
- 원본: `inbox/local-delegation-557/260512_3차일괄이양_의견조회_대상_특행포함_557개.xlsx`
- 시트: `전체(557개_411개+특행146개) (2)`
- 대상: 557개 사무
- 법률 기준: 189개
- 특행 포함 여부: 원본 `특행여부` 컬럼 보존

### 804 전체 기준

- 파일: `app/data/delegation-804.json`
- 기준: 기존 전체 804건 최종 F분류
- 용도: 전체 후보군 조망 및 557 의견조회 기준과 비교

## 현재 F분류

- **F1 형식상 전환후보**: 법률은 지자체 형식, 시행령은 비지자체·특행기관 등인 전환 검토군
- **F2 형식상 신설검토후보**: 법률은 지자체 형식이나 시행령 대응 수임 규정이 없거나 미확인인 신설 검토군
- **F3 직접권한·기위임**: 시행령 형식상 이미 지자체 직접권한/기위임으로 보이는 별도표기군
- **F5 형식연결 보완**: 법률-시행령 형식 연결은 있으나 조문명·대상·인용관계 확인이 필요한 보완군
- **F4 법령명 정규화**: 법령명 정규화 후 재매칭이 필요한 잔여군

## 생성 방식

1. Gmail 최신 수신 메일 제목 `557`에서 엑셀 첨부 다운로드
2. 엑셀 557개 행 파싱
3. 기존 804건 형식분류 결과와 `법령명+조항+사무명` 우선 매칭
4. 일부는 `법령명+조항` 단일 후보 매칭
5. 미매칭 항목은 보수적으로 F2 후속확인 대상으로 분류
6. 법제처 국가법령정보 API(law.go.kr) 현행 법령 조문을 가능한 범위에서 수록

## 원문 수록 현황

정적 JSON `app/data/delegation-557.json`, `app/data/delegation-804.json`에 법률·시행령 조문 데이터를 포함합니다.

- 법률 조문 참조: 557건
- 시행령 조문 참조: 230건
- 법률 원문 수록 법령: 185개
- 시행령 원문 수록 법령: 173개

일부 오기·폐지법령·시행규칙명·시행령 대응규정 미확인 항목은 후속 정규화/검토 대상입니다.

## 주요 화면

- 통합 매트릭스: 법률상 지자체 유형 × 시행령 현재 상태 × F분류
- 1차 매트릭스: 법률상 지자체 유형 × 시행령 수임자 형식
- 2차 매트릭스: 법률상 지자체 유형 × F분류
- 법률별 프로파일
- 필터 가능한 사무 목록
- 사무별 법률·시행령 원문 좌우 대조 패널

## 포지셔닝

이 프로젝트는 공픈클로(GongpunClaw)식 정책분석 산출물을 웹 탐색기로 바꾸는 실험입니다. 557개 의견조회 대상을 “사무 성격”이 아니라 “법률-시행령 형식” 기준으로 먼저 정렬해, 후속 실질 판단의 작업대를 만드는 것이 목적입니다.
